# Business Pattern Memory Index

## Purpose

Tracks recurring business patterns, client-specific behavior, seasonal effects, operational constraints, and demand signals.

This memory category helps the future engine adapt to each business model it serves.

## What Belongs Here

- seasonal demand behavior
- recurring demand spikes
- operational constraints
- business-specific habits
- client-specific staffing rules
- role-specific limitations
- known external triggers

## What Does Not Belong Here

- one-off forecast errors unless they reveal a pattern
- personal learning notes
- AI/model failures
- architecture decisions
- unverified assumptions

## Record Format

Each record should include:

- Date
- Business type
- Pattern observed
- Trigger
- Impact
- Confidence level
- Suggested rule or adjustment
- Approval status

## Suggested File Naming

```text
YYYY-MM-DD-business-pattern-name.md
```

Example:

```text
2026-06-22-restaurant-rainy-weekend-pattern.md
```
