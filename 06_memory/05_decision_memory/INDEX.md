# Decision Memory Index

## Purpose

Tracks important product, architecture, model, workflow, and workstation decisions.

This memory category prevents repeated debates and preserves why decisions were made.

## What Belongs Here

- architecture decisions
- product direction decisions
- tool parking/activation decisions
- model assignment decisions
- memory authority decisions
- governance decisions
- rejected alternatives
- consequences of major choices

## What Does Not Belong Here

- daily learning notes
- raw AI output
- unapproved ideas
- operational forecast results
- temporary thoughts with no decision

## Record Format

Each record should include:

- Date
- Decision
- Context
- Alternatives considered
- Reason
- Consequence
- Review date
- Status

## Suggested File Naming

```text
YYYY-MM-DD-decision-name.md
```

Example:

```text
2026-06-22-use-agents-rule-as-canonical.md
```
